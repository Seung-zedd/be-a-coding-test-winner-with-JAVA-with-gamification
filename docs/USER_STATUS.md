### 🧠 던전 통찰 (Dungeon Insights): Failure Rate

**1. 지문의 마법 (The Power of Reading)**

- "사용자가 현재 머물러 있는 스테이지 번호" $\rightarrow$ 이 짧은 문장에서 **(1) 배열 길이는 전체 유저 수**, **(2) 원소 값은 카운팅 배열의 인덱스**라는 두 가지 힌트를 동시에 추출함.

**2. 전력 분산 전술 (O(M+N))**

- 이중 `for`문 ($10^8$)의 함정을 피하기 위해, `stages`를 단 한 번 순회하며 각 층의 탈락자를 매핑하는 **Pre-counting** 기법 적용.

**3. 정렬의 지휘 (Complex Sorting)**

- `sort((e1, e2) -> { ... })` 블록 내에서 `int res = Double.compare(...)` 결과에 따라 분기 처리.
- **`res == 0` (비겼을 때)**의 다음 행동 강령(스테이지 번호 오름차순)을 명시하는 것이 핵심.
