### 🧠 던전 통찰 (Dungeon Insights): Dimensional Rifts

**1. 제약 조건의 비밀 (The Secrets of Constraints)**

- **$1 \le n \le 10^7$**: $n$ 자체는 `int` 범위지만, 격자 전체 크기($n^2$)는 $10^{14}$에 육박함. **메모리 초과(MLE)**를 피하기 위해 2차원 배열 선언은 절대 불가!
- **$0 \le left \le right < n^2$**: 인덱스 최대값이 $10^{14}$이므로, `left`와 `right` 그리고 이를 계산하는 인덱스 변수는 반드시 **`long`** 타입을 사용해야 함 (**오버플로우 오염 방지**).
- **$right - left < 10^5$**: 우리가 반환할 스냅샷의 크기는 고작 10만 개! 결과 배열은 `int[]`로 충분하며, 시간 복잡도 또한 $O(10^5)$ 내외로 해결 가능하다는 신호.

**2. 좌표 추출 마법 (Coordinate Mapping Logic)**

- **`row = k / n`**: $k$라는 전체 칸수를 '한 행의 길이($n$)'로 나누면, 내가 몇 번째 행에 도달했는지(몫)를 알 수 있음.
- **`col = k % n`**: 한 행의 길이를 채우고 남은 나머지는 현재 행에서의 위치(나머지)를 의미함.
- 🚫 **`n / k`를 하지 않는 이유**: 거꾸로 나누면 칸수 대비 행의 비율이라는 이상한 값이 나옴. 우리는 '그룹($n$)으로 나누어 묶음 번호(row)'를 찾는 것이 목적!
