# 🤖 AGENTS.md (보조 관전 기록 및 룰북)

이곳은 안티그레비티(Antigravity) 조수가 용사님의 전우로서 지켜야 할 철칙과 자동화 룰을 기록하는 곳입니다.

---

### 📜 자동화 룰 (Automation Rules)

**1. 승전보 브금(BGM) 자동 이사 룰**

- **Trigger:** 던전 클리어 랭크와 상관없이 새로운 '퀘스트 완료' 기록이 업데이트되었을 때.
- **Action:** `USER_STATUS.md` 내의 기존 승전보 브금 플레이어를 **가장 최근의 퀘스트 완료 기록 하단**으로 자동 이동시킨다.

**2. 지수적 복습 방어 룰 (Exponential Defense Rule)**

- **Trigger:** 수련 과제의 단계(Lv)가 상승할 때.
- **Action:** 다음 복습 예정일을 **[최초 학습일 + n일]** 공식에 따라 자동 산출한다.

**3. 브리핑 및 전장 자동 조정 룰 (Daily Briefing & Setup Rule)**

- **Trigger:** "오늘 해야 할 일 브리핑해줘" 또는 이와 유사한 요청 시.
- **Action:**
  1. `USER_STATUS.md`의 복습 주기와 `📖 성전(聖典)의 로드맵`을 분석하여 **오늘 정복해야 할 모든 퀘스트(복습 + 신규)**를 선별한다.
  2. 선별된 문제들에 대해서만 `TestSolution.java`에 전용 테스트 메서드를 생성/활성화하고, **그 외의 데드 테스트는 즉시 소거**한다.
  3. 용사님이 즉시 전투에 돌입할 수 있도록 요약된 전략 브리핑을 제공한다.
- **의도:** 용사님이 매일 전장을 정리하는 번거로움 없이, 조수(Agent)가 닦아놓은 전장에서 오직 효율적인 수련에만 매진하게 하기 위함.

**4. 전장 자동 세팅 룰 (Battlefield Auto-Setup Rule)**

- **Trigger:** 새로운 수련 과제(문제)에 진입할 때.
- **Action:**
  1. 해당 문제의 전용 클래스 파일과 `solution` 함수 껍데기만 생성한다. (구현부는 용사님의 몫으로 비워둔다)
  2. 백준 등 외부 플랫폼 문제라도 **프로그래머스(Programmers) 환경**을 기준으로 입출력을 자동 세팅한다. (매개변수로 입력받고 리턴값으로 출력)
  3. `TestSolution.java`에 해당 문제를 즉시 테스트할 수 있는 환경을 구축하며, **이전 문제의 테스트 메서드(Dead Method)는 깨끗이 제거하여 현재 문제에만 집중할 수 있게 한다.**
- **의도:** 용사님이 입출력 처리와 같은 부수적인 작업에 에너지를 낭비하지 않고 오직 핵심 로직 설계에만 집중하게 하기 위함.

**5. 퀘스트 랭크 산정 룰 (Quest Ranking Rule)**

- **Trigger:** 던전 클리어 후 승전 기록 업데이트 시.
- **Action:** 아래 기준에 따라 냉철하게 랭크를 산정한다.
  - **S-Rank:** 조수의 도움 없이 스스로 설계/구현/제출까지 완벽히 성공했을 때.
  - **A-Rank:** 알고리즘 설계는 맞았으나, 특정 로직 구현(문법 등)에서 조수의 도움을 받았을 때.
  - **B-Rank:** 알고리즘 설계 자체가 잘못되어 조수의 가이드를 받고 정답에 도달했을 때.
  - **C-Rank:** **문제의 지문 분석 자체가 틀려 출제자의 의도와 다른 방향으로 접근했을 때.**
- **의도:** 용사님의 현재 실력을 객관적으로 파악하고, 겸손한 자세로 수련에 정진하게 하기 위함.

**6. 던전 공략 기록 자동화 룰 (Quest Log Automation Rule)**

- **Trigger:** 던전 클리어(PASS) 직후 승전보 업데이트 시.
- **Action:**
  1. `USER_STATUS.md`의 **🗺️ 던전 공략 기록 (Quest Log)** 테이블 최상단에 오늘 날짜, 문제 이름, 산정된 랭크, 획득 XP(룰북 기준), 그리고 핵심 통찰(비고)을 즉시 기록한다.
  2. 기록 시, 테이블의 기존 데이터들은 아래로 밀어내어 최신 승전보가 항상 눈에 띄게 한다.
- **의도:** 용사님의 모든 전투 실적을 누락 없이 데이터화하여, 성장의 궤적을 한눈에 체감하게 하기 위함.

**7. 봉인된 Java 8 전장 룰 (Sealed Java 8 Battlefield Rule)**

- **Trigger:** 프로젝트 환경 설정 및 테스트 시.
- **Action:**
  1. `.classpath` 및 `.vscode/settings.json`을 **Java 8(1.8)**로 강제 고정하여 에디터 인텔리센스 단계부터 Java 8 문법만을 허용한다.
  2. 컴파일 시 `-source 8 -target 8` 옵션을 철저히 준수하여 프로그래머스 실전 환경과의 100% 동기화를 달성한다.
  3. `settings.json` 내에 확장 프로그램 엔진(`java.jdt.ls.java.home`)은 최신 JDK(21)로, 프로젝트 런타임(`java.configuration.runtimes`)은 **JDK 1.8** 경로(`C:\Program Files\Java\jdk-8.x`)로 이원화하여 명시하며, 버전 충돌 시 'Clean Language Server Workspace'를 수행한다.
- **의도:** 용사님이 실무/최신 문법이라는 달콤한 유혹에 빠지지 않고, 오직 코딩 테스트 현장의 척박한(Java 8) 환경에서도 살아남는 진정한 검사가 되게 하기 위함.

**8. 베테랑 자동완성 봉인 룰 (Veteran Autocomplete Seal Rule)**

- **Trigger:** **베테랑 우대(Veteran Exemption)**가 적용된 문제(최대 3회 복습)가 **2회차 복습** 이상에 진입했을 때.
- **Action:** 해당 과제를 수행하는 동안 VS Code의 모든 자동완성(IntelliSense) 기능을 강제로 비활성화한다.
- **의도:** 이미 숙달된 베테랑 문제에 대해서는 더 이른 시점에 자동완성을 봉인하여, 뇌와 손의 완벽한 동기화를 꾀하기 위함.

**9. 일반 자동완성 봉인 룰 (Standard Autocomplete Seal Rule)**

- **Trigger:** 일반 수련 과제(5회 복습)의 **복습 횟수가 3회차** 이상에 진입했을 때.
- **Action:** 해당 과제를 수행하는 동안 VS Code의 모든 자동완성(IntelliSense), 제안(Quick Suggestions), 매개변수 힌트 기능을 강제로 비활성화한다.
- **용어 정의:**
  - **[정복]**: 최초로 문제를 풀어낸 상태 (회차 계산에서 제외)
  - **[복습]**: 에빙하우스 주기(1, 3, 7, 14, 30)에 따른 재풀이
- **설정값:**
  - `"editor.suggestOnTriggerCharacters": false`
  - `"editor.quickSuggestions": { "other": false, "comments": false, "strings": false }`
  - `"editor.wordBasedSuggestions": "off"`
  - `"editor.parameterHints.enabled": false`
  - `"java.completion.enabled": false`
- **의도:** 자동완성이 없는 실제 코딩 테스트 환경에 대비하여, 머릿속의 로직을 손끝으로 완벽하게 인출하는 '인덱스의 지배자' 능력을 극대화하기 위함.

**9. 실시간 전황 정기 업데이트 룰 (Real-time Status Update Rule)**

- **Action:** 던전 클리어(정복 또는 복습 성공) 즉시 `USER_STATUS.md`를 업데이트한다.
- **필수 준수 사항:**
  - **[정복]**과 **[복습]** 태그를 엄격히 구분하여 기재한다.
  - 표(Table)의 마크다운 정렬이 어긋나지 않도록 정밀하게 수선한다.
  - 누적 XP와 레벨 진행도를 즉시 갱신한다.

**10. 최신 승전보 선포 룰 (Latest Victory Report Rule)**

- **Action:** 클리어 직후 `USER_STATUS.md` 하단의 `### 🎊 최신 승전보` 섹션에 용사님의 활약상을 한 줄 요약하여 선포한다.
- **형식:** `"🎊 퀘스트 완료: [랭크] 문제이름 [정복/복습 n회차]! 🏆"`

**11. 미세 팁 금지 룰 (No-Tip Reset Rule)**

- **Trigger:** "전장 초기화" 또는 이와 유사한 초기화 요청 시.
- **Action:** 클래스 파일 생성 및 초기화 시 주석으로 어떠한 힌트, 팁, 메서드명, 또는 사용해야 할 자료구조에 대한 언급도 일절 금지한다. 오직 문제의 핵심 정보(입출력, 제한 사항)만을 남긴다.
- **의도:** 용사님이 외부의 조언 없이 본인의 순수한 논리력과 기억만으로 전장을 헤쳐 나가게 하기 위함.

**12. 베테랑 우대 룰 (Veteran Exemption Rule)**

- **Trigger:** 용사님이 이미 훈련이 충분한(5회 이상 복습) 타 플랫폼/Kit 문제를 언급할 때.
- **Action:** 해당 퀘스트의 마스터링 조건을 기존 **5회 복습**에서 **복습 3회**로 파격 단축한다. 수련 테이블에도 `(Counts/3)` 형식으로 별도 관리한다.
- **의도:** 불필요한 단순 반복을 지양하고, 용사님이 새로운 고난도 전장에 더 빨리 투입될 수 있도록 전폭적으로 지원하기 위함.

**13. 브레인스토밍 및 수도코드 박제 룰 (Brainstorming & Pseudocode Preservation Rule)**

- **Trigger:** 용사님이 지문 분석, 시각화, 또는 수도코드를 작성한 브레인스토밍 스크린샷 위치를 알려주며 박제를 요청할 때.
- **Action:**
  1. 승전 여부와 상관없이, 용사님의 사고 과정이 담긴 성밀한 기록을 해당 섹션 폴더(예: `src/hash/images/`)로 복사하여 영구 보존한다.
  2. `USER_STATUS.md`의 해당 퀘스트 기록 하단에 상대 경로(`../src/[sector]/images/filename.png`) 형식으로 이미지를 박제한다.
- **의도:** 정답 코드보다 더 가치 있는 **'용사님의 사고의 발자취(Brainstorming)'**를 보존하여, 논리의 근육이 성장하는 과정을 증명하기 위함.
