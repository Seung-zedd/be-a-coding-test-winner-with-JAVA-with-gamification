# ⚔️ 코딩 테스트 정복 가이드 (ConquerTheCode Guide) ⚔️

이 문서는 용사(USER)와 부관(AI)이 협력하여 코딩 테스트라는 던전을 공략하기 위한 **전투 교본**입니다. 우리는 아래와 같은 특수 주석 시스템을 사용하여 효율적으로 소통합니다.

---

### 🛡️ 주석 시스템 (Commenting System)

| 기호        | 역할             | 설명                                                                                        |
| :---------- | :--------------- | :------------------------------------------------------------------------------------------ |
| `/* ... */` | **GM 힌트**      | 부관(AI)이 제공하는 정답에 가까운 결정적인 힌트입니다. 에디터에서 하이라이트 표시가 됩니다. |
| `//`        | **로직 설계**    | 용사님이 실제 코드를 작성하기 전, 생각의 흐름을 정리하는 의사코드(Pseudo-code)입니다.       |
| `//?`       | **복습 포인트**  | 문제를 풀고 난 뒤, 다시 볼 때 이해가 안 가거나 의문이 드는 점을 기록합니다.                 |
| `//!`       | **경고 및 실수** | 이전에 저질렀던 실수나, 이 알고리즘에서 가장 주의해야 할 함정을 표시합니다.                 |

### 📜 실전 예시 (Example: n^2 배열 자르기)

`TestSolution.java`에서 발췌한 실제 전투 준비 코드입니다.

```java
public class TestSolution {
    public int[] solution(int n, long left, long right) {
        // [재도전 퀘스트: 내일의 나에게]
        // 1. 결과 배열 크기 설정 (right - left + 1)
        // 2. 반복문 제어 (결과 배열 index 0부터 len-1까지)
        // 3. 실제 인덱스 k 소환 (left + i)
        // 4. n차 격자 좌표 매핑 (row = k / n, col = k % n)
        // 5. 규칙 적용 (Math.max(row, col) + 1)

        //? [비밀 단서 1] 세상 전체의 크기(n^2)는 long이지만, 한 줄의 길이(n)는 int 범위임을 기억하라.
        //? [비밀 단서 2] 보물은 전체 지도(k번지)에 있지만, 내 가방(arr)은 0번(i)부터 채워야 한다.

        return null;
    }
}
```
